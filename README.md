# E-Commerce SQL Analysis

## Overview
This project analyzes an e-commerce dataset using SQL to answer real business questions around revenue, customer behavior, product performance, and sales trends.

## What I Analyzed
- Total revenue performance
- Top products and categories
- Customer segments (age groups)
- Order and revenue trends over time
- Payment method usage
- Repeat vs one-time customers
- Product volume vs revenue

## Key Insights
- A small number of products and categories drive most of the revenue
- Certain customer groups contribute more to total sales than others
- Revenue and order volume change over time, showing trends and possible seasonality
- Payment methods vary in both usage and revenue contribution
- There is a mix of one-time and repeat customers, showing opportunities for retention
- Some products sell a lot but generate less revenue, while others sell less but bring in more value

## Skills Demonstrated
- SQL aggregations (SUM, COUNT, AVG)
- GROUP BY and HAVING
- Common Table Expressions (CTEs)
- Conditional logic (CASE WHEN)
- Time-based analysis
- Business-focused data interpretation

## Tools Used
- SQL (SQL Server)
- CSV dataset
